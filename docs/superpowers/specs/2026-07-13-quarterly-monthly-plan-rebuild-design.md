# 季度里程碑计划与月度项目经营计划重建设计

## 1. 设计目标

计划管理只保留两个互相独立的业务模块：

- 季度里程碑计划
- 月度项目经营计划

季度计划与月度计划当前不建立父子、来源、衍生、转换或其他关联关系。新模块以 `doc/技术中心2026年第三季度里程碑计划.xlsx`、`doc/技术中心月度（7月）项目经营计划.xlsx` 及用户提供的表单截图为字段和交互依据，替换现有复杂的通用计划模型。

## 2. 模块结构

- 保留统一菜单“计划管理”。
- `/plans` 作为统一列表入口，提供“季度里程碑计划”和“月度项目经营计划”两个页签。
- 两类计划分别拥有独立的新建、详情、编辑页面、数据结构和业务校验。
- 删除年度计划、半年度计划、计划总览、计划外事项、季度生成月度计划、通用计划表单和关联计划选择。
- 删除季度与月度之间，以及计划与需求、项目、任务、产品版本之间的关联设计。

## 3. 权限规则

- 产品线小组长可以新建、编辑、保存草稿和发布本组计划。
- 产品线普通成员只能查看本组计划。
- 系统管理员可以查看和维护全部小组计划。
- 一名组长只负责一个产品线小组时，新建计划自动带出该小组。
- 一名组长负责多个产品线小组时，由组长人工选择归属小组。
- 不属于任何产品线小组且不是管理员的用户不能新建计划。

## 4. 通用发布规则

- 计划状态只保留“草稿”和“已发布”。
- 草稿可以自由编辑和删除。
- 已发布计划仍允许对应产品线小组长或管理员编辑，系统记录最后修改人和最后修改时间。
- 已发布计划不能直接删除；管理员需要先退回草稿，之后才能删除。
- 本阶段不建设审批流、发布版本历史和发布后变更记录表。
- 每次发布均执行完整业务校验；草稿允许暂存未完成的新增行。

## 5. 季度里程碑计划

### 5.1 计划头部

- 归属产品线小组：必填。
- 计划年度：必填。
- 所属季度：必填。
- 计划状态：草稿、已发布。
- 制定人、创建时间、发布时间、最后修改人、最后修改时间：系统记录。
- 同一产品线小组、同一年度、同一季度只能存在一份季度计划。

### 5.2 季度目标看板

每份季度计划包含零到多条季度目标。目标行字段如下：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 序号 | 系统生成 | 支持人工排序 |
| 目标域 | 是 | 产品迭代、产品交付、产品调研、市场支持、运维稳定、技术创新、AI赋能 |
| 季度目标 | 是 | 多行文本 |
| 成功标准 | 是 | 验收或成功标准，多行文本 |
| M1 目标 | 是 | 季度第一个月目标 |
| M1 状态 | 是 | 目标状态字典 |
| M2 目标 | 是 | 季度第二个月目标 |
| M2 状态 | 是 | 目标状态字典 |
| M3 目标 | 是 | 季度第三个月目标 |
| M3 状态 | 是 | 目标状态字典 |
| 当前完成情况 | 否 | 人工填写实际完成内容 |
| 达成率 | 是 | 人工填写 0–100%，默认 0 |
| 季度状态 | 是 | 目标状态字典 |
| 关键依赖 | 否 | 多行文本 |
| 备注 | 否 | 多行文本 |

目标状态固定为：未开始、进行中、已完成、延期风险、已延期、暂缓。

季度计划发布时至少需要一条完整目标；风险登记允许为空。

### 5.3 季度交付风险登记册

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 风险编号 | 系统生成 | 当前计划内按 `R1、R2……` 编号 |
| 风险描述 | 是 | 风险内容 |
| 影响里程碑 | 否 | 自由文本，不关联其他对象 |
| 发生概率 | 是 | 高、中、低 |
| 影响程度 | 是 | 高、中、低 |
| 综合风险级 | 是 | 高、中、低 |
| 触发条件 | 否 | 多行文本 |
| 应对策略 | 是 | 多行文本 |
| 预警节点 | 否 | 自由文本 |
| 当前状态 | 是 | 未触发、已触发、处理中、已关闭 |

### 5.4 季度表单交互

- 页面头部维护年度、季度和归属产品线小组。
- “季度产品目标看板”和“季度交付风险登记册”以两个可横向滚动的行内编辑表格展示。
- 支持新增目标行、新增风险行、行内编辑、单行删除和排序。
- 页面顶部和底部均提供返回、保存为草稿、制定并发布操作。

## 6. 月度项目经营计划

### 6.1 计划头部

- 归属产品线小组：必填。
- 计划年度、月份：必填。
- 计划状态：草稿、已发布。
- 制定人、创建时间、发布时间、最后修改人、最后修改时间：系统记录。
- 同一产品线小组、同一年度、同一月份只能存在一份月度计划。

### 6.2 八个业务板块

月度计划包含八个独立板块，各板块支持零到多行数据。

1. 产品交付计划
   - 模块/功能/版本
   - 交付内容
   - 计划完成时间
2. 项目交付计划
   - 项目名称
   - 交付内容
   - 计划完成时间
   - 客户名称，可选
3. 市场化动作计划
   - 产品/项目
   - 市场化动作
   - 输出成果
   - 计划完成时间
4. 成本优化计划
   - 优化项
   - 当前问题
   - 优化措施
5. AI+赋能产品计划
   - 事项
   - 输出成果
   - 计划完成时间
6. AI+效能提升计划
   - 事项
   - 输出成果
   - 计划完成时间
7. 项目风险
   - 风险项
   - 风险等级：高、中、低
   - 影响范围
   - 应对措施
8. 资源需求
   - 需求类型：人员、技术、预算、管理协调
   - 具体内容
   - 紧急程度：高、中、低
   - 需要支持部门/小组

### 6.3 月度校验与交互

- 八个板块本身都允许为空。
- 一旦新增某行，除“客户名称”外，该行定义的业务字段都必须完整填写。
- 计划完成时间在产品交付、项目交付、市场化动作、AI+赋能产品和 AI+效能提升板块中必填。
- 成本优化、项目风险和资源需求不设置计划完成时间。
- 草稿可以暂存不完整行。
- 发布时至少一个板块包含一条完整数据，并校验所有已有行。
- 每个板块采用独立的行内编辑表格，支持新增、编辑、删除和排序。
- 月度字段均为本期自由文本，不关联需求、项目、任务或产品版本。

## 7. 列表与详情

### 7.1 统一列表

- 顶部使用季度、月度两个页签。
- 支持年度、季度或月份、产品线小组、状态筛选。
- 小组长默认查看本组；管理员可以查看全部小组。
- 两个页签分别提供新建按钮。

季度计划列表字段：

- 年度季度
- 产品线小组
- 小组长
- 目标数量
- 风险数量
- 计划状态
- 发布时间
- 最后修改时间
- 操作

月度计划列表字段：

- 年度月份
- 产品线小组
- 小组长
- 八板块事项总数
- 风险数量
- 资源需求数量
- 计划状态
- 发布时间
- 最后修改时间
- 操作

### 7.2 详情页

- 详情页按对应表单结构只读展示。
- 季度详情展示目标看板和风险登记册。
- 月度详情展示全部八个板块；空板块显示“暂无数据”。
- 不展示季度与月度关系图、来源计划或关联计划。

## 8. 数据字典边界

本阶段不建设通用数据字典管理页面，先集中治理散落选项。

### 8.1 固定系统状态

以下值参与程序校验和流程控制，不允许后台动态修改：

- 计划状态：草稿、已发布
- 目标状态：未开始、进行中、已完成、延期风险、已延期、暂缓
- 风险状态：未触发、已触发、处理中、已关闭

### 8.2 集中业务选项

以下选项统一在计划模块的单一字典定义中维护，表单、列表和详情共享：

- 季度目标域
- 风险等级
- 资源需求类型
- 紧急程度

当业务选项频繁变化、需要按产品线配置，或至少三类字典明确要求管理员自行维护时，再建设“基础数据管理”页面。未来页面只开放业务选项，不开放系统流程状态。

## 9. 旧模块与数据处理

- 不迁移现有计划数据。
- 新模块上线时删除旧计划、计划工作项和计划阶段数据。
- 删除旧计划模型及需求、项目、任务、产品版本中指向旧计划工作项的关系字段，但不删除需求、项目、任务或产品版本本身。
- 删除旧计划页面、动作、校验、菜单入口和旧数据字典残留。
- 第一阶段支持人工重新录入。
- Excel 导入作为后续独立功能，不纳入本次重构。

## 10. 数据模型边界

新数据模型使用独立聚合：

- `QuarterlyPlan`
- `QuarterlyGoal`
- `QuarterlyRisk`
- `MonthlyPlan`
- 月度八类独立明细表

季度和月度计划只共同引用产品线小组、制定人和最后修改人，不互相引用，不共用通用明细表。

## 11. 错误处理

- 服务端对所有写操作重新鉴权，不能依赖页面是否显示按钮。
- 唯一周期冲突返回“该产品线小组在当前周期已存在计划”。
- 保存草稿失败时保留页面输入并显示明确错误。
- 发布失败时定位到具体板块和行，禁止部分发布。
- 删除已发布计划直接拒绝，管理员退回草稿后才能删除。

## 12. 验证与验收

- 验证小组长、普通成员和管理员权限。
- 验证同一小组同一周期唯一性。
- 验证草稿宽松保存和发布完整校验。
- 验证季度目标、风险及月度八板块的增删改排序。
- 验证季度达成率范围和各固定状态值。
- 验证已发布计划编辑、退回和删除限制。
- 验证旧菜单、旧路由、旧关系、旧模型和旧数据字典全部清除。
- 验证需求、项目、任务、产品版本数据未因删除计划关系而被删除。
- 验证 Prisma、TypeScript、目标 lint、生产构建和关键路由运行状态。
