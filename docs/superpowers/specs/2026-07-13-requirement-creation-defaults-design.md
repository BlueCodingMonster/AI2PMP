# 需求创建时间、内容简述与产品线默认值设计

## 目标

在现有精简需求池基础上补充三个录入规则：创建时间可维护、需求内容可简述、产品线在条件明确时自动默认。

## 字段与交互

- `createdAt` 为必填日期时间字段。
- 新建需求时，表单默认填入当前本地日期和时间，用户保存前可以修改。
- 编辑需求时，表单显示数据库中的创建时间，允许修改但不允许清空。
- 新增 `summary` 可空多行文本字段，界面名称为“需求内容简述（可选）”。
- 详情页展示需求内容简述；列表不增加该列，保持现有表格宽度。

## 产品线默认规则

新建页根据当前登录人的固定产品线成员关系计算默认值：

- 恰好属于一个产品线小组时，默认选中该小组。
- 属于多个产品线小组时，不设置默认值。
- 不属于任何产品线小组时，不设置默认值。
- 产品线始终是可选项，用户可以切换到其他小组或清空选择。

编辑已有需求时只使用需求当前保存的产品线，不重新套用登录人的默认小组。

## 数据与校验

- `Requirement` 新增可空的 `summary` 文本列。
- `createdAt` 沿用现有数据库列，但纳入创建和更新输入。
- 前端阻止空创建时间提交，服务端校验创建时间必填且能够解析为有效日期。
- 服务端创建与更新都显式保存 `createdAt`，避免只依赖浏览器校验。
- 现有记录的数据保持不变；迁移只新增 `summary`。

## 范围边界

- 不把内容简述加入需求列表列或搜索范围。
- 不改变产品线成员关系模型，也不创建“主产品线”概念。
- 不改变需求状态、评审时间及计划/任务的可选关联规则。

## 验证

- 契约测试覆盖 `summary`、创建时间必填及读写、单一产品线默认规则。
- Prisma schema 与迁移验证通过。
- TypeScript、目标文件 ESLint 和生产构建通过。
- 运行态检查新建、编辑和详情路由。
