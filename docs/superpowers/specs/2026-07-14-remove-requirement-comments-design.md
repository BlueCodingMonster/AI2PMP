# 删除需求讨论与备注设计

## 目标

从需求详情中彻底移除“讨论与备注”，并删除需求评论对应的数据结构和数据。

## 范围

- 删除需求详情页的讨论与备注区域。
- 删除需求评论前端组件和新增需求评论服务动作。
- `Requirement` 模型删除 `comments` 关系。
- 通用 `Comment` 模型删除 `requirementId` 字段及对应关系和索引。
- 数据库迁移删除所有 `requirementId` 非空的评论，再删除该字段、外键和索引。

## 保留边界

- 保留通用 `Comment` 表。
- 保留任务评论和缺陷评论的字段、关系、动作、组件及数据。
- 不修改需求的其他字段、可选任务关联和统一只读表单。

## 当前数据

实施前数据库中需求评论为 0 条。迁移仍显式删除需求评论，以保证其他环境存在历史数据时可安全执行。

## 验收

- 需求详情不显示“讨论与备注”。
- 代码中不存在需求评论组件或新增需求评论动作。
- Prisma 中不存在 `Requirement.comments`、`Comment.requirementId` 或需求评论索引。
- 任务和缺陷评论保持可用。
- 数据库迁移、需求契约、Prisma 校验、类型检查和构建通过。
